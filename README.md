## What is Headless Shopify?

Headless Shopify architecture decouples the frontend (what customers see) from the backend (Shopify's commerce engine). This allows brands to:

- ‚ö° **Achieve faster load times** - Custom frontends optimized for performance
- üé® **Full design freedom** - No theme limitations
- üì± **Omnichannel experiences** - Same backend, multiple frontends
- üéØ **Advanced personalization** - Tailored shopping experiences
- üåç **Global scalability** - Handle high traffic with custom infrastructure

## Tech Stack Examples

Common technology combinations used in headless Shopify implementations:

### Shopify Hydrogen + Oxygen
```
Frontend: Shopify Hydrogen (React-based)
Hosting: Shopify Oxygen (free, global CDN)
Backend: Shopify Storefront API
Used by: Allbirds, SKIMS, Gymshark, Lady Gaga
```

### Next.js + Vercel
```
Frontend: Next.js
Hosting: Vercel
Backend: Shopify Storefront API
CMS: Sanity / Contentful
Used by: Magda Butrym, Lift Foils, AETHER
```

### Next.js + Netlify + Contentful
```
Frontend: Next.js
Hosting: Netlify
Backend: Shopify Storefront API
CMS: Contentful
Used by: Victoria Beckham Beauty
```

### Gatsby + Netlify
```
Frontend: Gatsby
Hosting: Netlify
Backend: Shopify Storefront API
Used by: Various smaller brands
```

### Custom React + Cloudflare
```
Frontend: Custom React
Hosting: Cloudflare Workers
Backend: Shopify Storefront API
Used by: Enterprise brands
```

## Contributing

Contributions are welcome! Please read the contribution guidelines first.

### How to Contribute

1. Fork this repository
2. Create a new branch: `git checkout -b add-new-store`
3. Add your store to the appropriate category
4. Commit your changes: `git commit -am 'Add StoreName'`
5. Push to the branch: `git push origin add-new-store`
6. Submit a pull request

### Contribution Guidelines

- ‚úÖ **Verify the store is headless** - Should use Shopify as backend with a custom frontend
- ‚úÖ **Include accurate information** - Store name, description, and working URL
- ‚úÖ **Follow the format** - Match the existing table structure
- ‚úÖ **One store per PR** - Makes review easier
- ‚ùå **No affiliate links** - Keep links clean and direct

### Criteria for Inclusion

| Requirement | Description |
|-------------|-------------|
| ‚úÖ Shopify Backend | Uses Shopify/Shopify Plus as commerce engine |
| ‚úÖ Custom Frontend | Has a decoupled/headless frontend |
| ‚úÖ Live Store | Store is publicly accessible |
| ‚úÖ Quality Implementation | Demonstrates good headless practices |

### How to Verify a Store is Headless

1. Check the technology using [BuiltWith](https://builtwith.com/) or [Wappalyzer](https://www.wappalyzer.com/)
2. Look for React/Next.js/Gatsby in the source code
3. Check if Shopify checkout is on a different domain
4. Verify fast client-side navigation between pages
